# SEO Content Audit

Ты анализируешь контент и семантику сайта. Пишешь результат в `<project>/research/content-audit.md`.

## Эталон формата

Формат аналогичен `factor/research/gap-analysis-results.md` и `factor/research/competitive-analysis-kev.md` (разделы, таблицы, кластеры). Прочитай эти файлы перед началом.

## Input contract

- `<project>/data/brief.md` заполнен (ниша, регион, ЦА, продукт)
- `<project>/research/tech-seo-audit.md` опционально — оттуда берём список страниц сайта

## Шаги

### 1. Инвентаризация существующего контента

- Получи sitemap или список URL сайта (через WebFetch)
- Сгруппируй по типам: главная, услуги/продукты, блог, О компании, кейсы, прочее
- Для ключевых страниц (10-20 шт) собери через WebFetch: URL, title, h1, первый абзац, длину текста
- Запиши в таблицу

### 2. Семантическое ядро — первичный сбор

**Источники:**
- `seo-competitors-gap` скилл (если запущен до тебя) — его keys.so-данные
- WebSearch по базовым запросам ниши → смотрим автосаджесты, "люди также спрашивают"
- brief.md → оттуда берём базовые термины продукта

Собери 50-200 релевантных запросов (не гонись за количеством — важна релевантность).

Для каждого запроса: фраза, примерная частота (если известна), интент (I/N/T/C — информационный/навигационный/транзакционный/коммерческий).

### 3. Кластеризация

Сгруппируй запросы в 10-20 кластеров по смыслу. Для каждого кластера:
- Название кластера
- Ключевая фраза
- Количество запросов
- Суммарная частота
- Тип интента
- Под какую страницу сайта подходит (URL) или "нужна новая страница"

### 4. Покрытие (gap анализ по текущему сайту)

Для каждого кластера проставь:
- ✅ Есть страница, хорошо оптимизирована
- 🟡 Есть страница, но не оптимизирована под кластер
- ❌ Нет страницы

### 5. Анализ топ-5 SERP для приоритетных кластеров

Возьми топ-3 кластера (высокая частота + бизнес-релевантность) и для каждого:
- Получи топ-5 результатов Яндекса через WebSearch
- Для каждого результата: домен, тип страницы, примерный объём, формат (статья/лендинг/каталог)
- Выведи паттерн: "в топе — лендинги сервисов" / "в топе — статьи по 3000+ слов"

Это определяет формат контента, который нужно делать.

### 6. Контент-план рекомендаций

Составь список страниц/статей для создания:
- Приоритет (high/med/low по impact × effort)
- Тип (новая услуга/лендинг, новая статья блога, переработка существующей)
- Кластер, под который пишется
- Целевой запрос H1
- Краткий бриф (2-3 предложения: про что, какой угол)

## Интерактивные остановки

1. **После шага 2 (ядро).** Покажи топ-20 фраз, спроси: **"Ядро релевантно? Что добавить/убрать?"**
2. **После шага 3 (кластеры).** Покажи все 10-20 кластеров, спроси: **"Какие 5 кластеров в приоритете для SERP-анализа?"**
3. **После шага 6.** Покажи топ-10 идей контент-плана, спроси: **"Сохраняю или корректируем?"**

## Definition of Done

- [ ] Инвентаризация текущего контента сделана (минимум 10 страниц проанализировано)
- [ ] Семантическое ядро собрано (≥50 фраз) и размечено по интентам
- [ ] Кластеризация с покрытием (✅/🟡/❌)
- [ ] SERP-анализ для топ-3 приоритетных кластеров
- [ ] Контент-план с приоритетами и брифами
- [ ] Нет пустых секций, нет `[ПРОВЕРИТЬ]` в высокоприоритетных рекомендациях
- [ ] Формат таблиц согласован с эталонами factor/research/

## Gotchas

- **Частоту не придумывать** — если данных keys.so нет, писать «частота неизвестна» или «по автосаджесту». Не ставить цифры из головы.
- **Три интерактивных стопа — обязательны** — нельзя пройти все 6 шагов без остановок. Пользователь должен подтвердить ядро и кластеры перед дальнейшей работой.
- **Ниша и регион из brief.md, не из домена** — `.yaroslavl.ru` не означает Ярославль, `.spb.ru` не означает только Петербург.
- **Статус покрытия — на основе реального контента** — перед тем как ставить ✅, прочитать страницу через WebFetch и проверить, что там действительно есть текст под этот кластер.
- **Контент-план — конкретный H1, не тема** — бриф «написать про CRM» не годится. Нужен конкретный H1 с ключом и углом.
- **Информационные кластеры без транзакционной связки — low priority** — трафик ≠ лиды. Если кластер не приводит к контакту — он не высокоприоритетный для бизнеса.

## Анти-паттерны

- ❌ Гнаться за объёмом ядра (200+ фраз) в ущерб релевантности
- ❌ Писать "информационный трафик" без привязки к транзакционным кластерам (бизнес = деньги, не трафик)
- ❌ Кластеры в отрыве от фактических страниц сайта
- ❌ Бриф статьи без формата и объёма
- ❌ Игнорировать brief.md — ниша и ЦА задают фильтр релевантности
