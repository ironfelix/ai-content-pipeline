# Audit Report Builder

Ты собираешь HTML-отчёт по результатам маркетинг-аудита. Читаешь все `<project>/research/*.md`, склеиваешь в единый документ с навигацией, применяешь стили, сохраняешь в `<project>/reports/audit-report.html`.

## Когда запускать

- После того как `marketing-audit` (или его под-скиллы) закончил работу и все `research/*.md` созданы
- Когда пользователь явно просит «сделай отчёт в HTML», «собери отчёт для клиента», «надо показать заказчику»

## Input contract

- Существует `<project>/research/audit-summary.md` — это **главный** документ, из которого берётся hero и top-секции
- Существуют (в любом количестве) дополнительные `<project>/research/*.md` — попадут в отдельные секции
- Существует `<project>/data/brief.md` — для hero-блока (название клиента, URL, дата)
- Существует папка `<project>/reports/`

Если `audit-summary.md` нет — не собирай отчёт, скажи пользователю запустить `marketing-audit` сначала.

## Шаги

### 1. Определи проект

Спроси у пользователя (если не очевидно из контекста) — для какого проекта собираем отчёт. Ожидаемый путь: `ai-seo/<project>/`.

### 2. Проверь Input contract

- Проверь, что `research/audit-summary.md` существует
- Перечисли все `.md` файлы в `research/`
- Прочитай `data/brief.md` — вытащи название клиента и URL

### 3. Запусти build_report.py

```bash
python3 /Users/ivanilin/.claude/skills/audit-report-builder/scripts/build_report.py <путь_к_проекту>
```

Скрипт:
- Читает все `research/*.md`
- Парсит через python `markdown` библиотеку
- Применяет HTML-шаблон
- Сохраняет в `reports/audit-report.html`

### 4. Верифицируй результат

- Проверь размер файла (должен быть > 10 KB, иначе что-то пошло не так)
- Покажи пользователю путь и предложи открыть в браузере: `open <путь>`

### 5. Опционально — кастомизация

Если пользователь просит изменить:
- **Цвет акцента** → передай `--accent "#hex"` в build_report.py
- **Название клиента** → передай `--client "Название"`
- **Скрыть какие-то секции** → правь напрямую в `audit-report.html` после генерации (быстрее, чем править скрипт)

## Output contract

Создаётся один файл: `<project>/reports/audit-report.html`

Структура отчёта:
1. **Hero** — название клиента, URL, дата, скоуп
2. **TL;DR** — первый раздел audit-summary.md (топ-5 проблем + топ-10 точек роста)
3. **Навигация** — sticky TOC справа или сверху
4. **Секции** — каждый research/*.md как отдельная секция с заголовком
5. **Footer** — ссылки на исходные markdown-файлы, дата генерации

## Зависимости

```bash
pip3 install --user markdown pygments
```

Проверь наличие перед первым запуском.

## Definition of Done

- [ ] Файл `<project>/reports/audit-report.html` создан
- [ ] Размер > 10 KB
- [ ] Пользователь видел путь и знает, как открыть
- [ ] В hero правильные название клиента и URL из brief.md
- [ ] Все research-файлы попали в отчёт (ни один не потерян)

## Анти-паттерны

- ❌ Генерировать отчёт, если `audit-summary.md` отсутствует — он является главным источником
- ❌ Встраивать внешние JS-библиотеки — отчёт должен открываться офлайн, с одного файла
- ❌ Копировать текст из markdown вручную — всегда через build_report.py
- ❌ Использовать фирменные цвета Фактора Продаж по умолчанию — они только для factor/ проекта
