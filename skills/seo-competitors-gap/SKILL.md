# SEO Competitors & Gap Analysis

Ты находишь органических конкурентов сайта и делаешь gap-анализ через keys.so API. Пишешь результаты в `<project>/research/gap-analysis.md` и (опционально) `<project>/research/competitive-analysis-<topic>.md`.

## Эталон формата

**ОБЯЗАТЕЛЬНО прочитай** перед началом:
- `/Users/ivanilin/Documents/ivanilin/ai-seo/factor/research/gap-analysis-results.md` — формат gap-анализа
- `/Users/ivanilin/Documents/ivanilin/ai-seo/factor/research/competitive-analysis-kev.md` — формат детального анализа топ-5 по конкретному запросу

Твой output повторяет эти форматы.

## Input contract

- `<project>/data/brief.md` заполнен (URL, ниша, регион)
- `KEYSO_API_TOKEN` в `ai-seo/.env` и он РАБОЧИЙ (вернёт не 401)
- База keys.so определена (обычно Яндекс.Россия, но см. brief для региона)

## Pre-flight check

Перед началом вызови `scripts/keyso_client.py check` — это проверит что токен живой. Если 401 — останови аудит, скажи пользователю обновить токен в `.env` и не продолжай.

## Шаги

### 1. Профиль домена

```
python3 scripts/keyso_client.py profile <domain>
```

Получи: vis, ключи в топ-1/3/5/10/50, количество страниц в индексе, DR.

### 2. Список конкурентов

```
python3 scripts/keyso_client.py competitors <domain> --limit 30
```

Получи топ-30 доменов по пересечению семантики.

### 3. Фильтрация конкурентов

Из списка отфильтруй:
- Технологические платформы широкого профиля (если они не релевантны нише) — bitrix, битрикс24, kokoc и т.п.
- Агрегаторы, если ниша не предполагает конкуренцию с ними
- Маркетплейсы и классифайды, если не релевантны

Оставь 5-10 тематически близких.

**ОСТАНОВКА:** Покажи пользователю отфильтрованный список и спроси: **"Использую этих 5-10 для gap-анализа, или скорректируем?"**

### 4. Gap-анализ

Для подтверждённых конкурентов:
```
python3 scripts/keyso_client.py gap <our_domain> <competitor1> <competitor2> ...
```

Получи ключи, которые есть у конкурентов и нет у нас (или у нас в топ-50+, а у них в топ-10).

### 5. Кластеризация gap-ключей

Сгруппируй gap-ключи в 10-15 смысловых кластеров. Для каждого:
- Название
- Количество ключей
- Суммарная частота
- У каких конкурентов есть (сколько из N)
- Тип интента
- Комментарий: "есть смысл брать" / "не наша ниша" / "требует нового продукта"

### 6. Топ-3 приоритетных направления

Выбери 3 кластера, которые:
- Бизнес-релевантны (приводят клиента, а не абстрактный трафик)
- Достижимы (в топе не монстры с DR 80+)
- Совпадают с целями в brief.md

### 7. Детальный SERP-анализ приоритетных кластеров (опционально)

Для каждого из топ-3 кластеров — создай отдельный файл `competitive-analysis-<topic>.md` с разбором топ-5 выдачи по главному запросу кластера (формат см. эталон `competitive-analysis-kev.md`):
- Домен
- Заголовок
- Примерный объём текста
- Формат (лендинг/статья/каталог)
- Сильные/слабые стороны
- Что взять себе

## Definition of Done

- [ ] Профиль домена получен
- [ ] Список конкурентов получен и отфильтрован с обоснованием
- [ ] Пользователь подтвердил список конкурентов
- [ ] Gap-ключи собраны и кластеризованы
- [ ] Топ-3 приоритетных направления выделены и обоснованы
- [ ] Нет домыслов, только данные из keys.so с источником
- [ ] Формат совпадает с эталоном

## Gotchas

- **Pre-flight check обязателен** — без `keyso_client.py check` начинать нельзя. 401 в середине анализа = потеря времени и данных.
- **factor/research/* — эталон формата, не контент** — нельзя брать выводы из gap-analysis-results.md Фактора и переносить в другой проект. Только структура таблиц и разделов.
- **Стоп после фильтрации конкурентов — обязателен** — нельзя идти к gap без подтверждения списка пользователем. Показать список и ждать.
- **Регион из brief.md, не из домена** — `.yaroslavl.ru` не означает Ярославль.
- **Информационные кластеры в топ-3 — только с обоснованием** — трафик без транзакционной ценности не является точкой роста для бизнеса, если явно не задано в целях brief.
- **Не называть конкурентов "нерелевантными" без проверки** — проверить через WebFetch хотя бы главную страницу, прежде чем исключить домен.

## Анти-паттерны

- ❌ Не проверять токен → падение посреди анализа
- ❌ Брать всех конкурентов без фильтрации → мусор в gap
- ❌ "Информационные" кластеры без бизнес-ценности в топ-приоритете
- ❌ Копировать обоснования из factor/research/* (это разные ниши)
- ❌ Гнаться за объёмом gap-ключей вместо релевантности
