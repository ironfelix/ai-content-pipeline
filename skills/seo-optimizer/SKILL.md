# SEO Optimizer — финальная SEO-упаковка статьи

## Шаг 0 — Загрузить конфиг проекта

**Определить проект** — по аргументу `project=` или дефолт `factor`:

```python
import os

# project= аргумент, если передан (напр. project=crmgroup)
# Иначе — "factor"
project_dir = "<значение аргумента project= если передан, иначе 'factor'>"

config_path = os.path.join(project_dir, "project.md")
if os.path.exists(config_path):
    with open(config_path) as f:
        project_config = f.read()
    print(f"Проект: {project_dir} → {config_path}")
else:
    project_config = None
    print(f"project.md не найден в {project_dir}/ → используем дефолты Фактора")
```

Если `project.md` найден — использовать его значения:
- `brand_name` — название бренда для SEO Title и Description
- `internal_links_map` — карта внутренних ссылок (вместо `factor/data/company-context.md` раздел 9)
- `company_context` — контекст компании для внутренних ссылок
- `search_engine` — целевой поисковик (дефолт: Яндекс)

Если `project.md` НЕ найден → фолбэк на дефолты Фактора:
- `brand_name` = «Фактор Продаж»
- `internal_links_map` = `factor/data/company-context.md` раздел 9
- `search_engine` = Яндекс

Если статья не передана — спроси: «Передай текст статьи и целевой ключевой запрос.»

## Шаг 1 — H1

Правила H1:
- Целевой запрос буквально + год
- Длина: 50–70 символов
- Без украшений и «умных» слов
- Год обязателен (повышает CTR в выдаче)
- **Год брать из `memory/MEMORY.md` раздел `# currentDate`** — не писать наугад

Предложи **3 варианта H1** — пусть пользователь выберет:
```
Вариант 1: [буквальный ключ + год]
Вариант 2: [ключ + уточнение + год]
Вариант 3: [ключ с другим порядком слов + год]
```

## Шаг 2 — SEO Title

Правила:
- 50–60 символов (считай символы!)
- Целевой ключ в начале
- Год — желательно
- Без «|» если влезает и без разделителей

Пример:
```
❌ «Как провести аудит отдела продаж | Фактор Продаж»  (56 — ок, но слабо)
✅ «Аудит отдела продаж 2025: что проверять и как»     (48 — отлично)
```

Предложи **2 варианта** с подсчётом символов.

## Шаг 3 — SEO Description

Правила:
- 120–155 символов (считай!)
- Структура: ключ + ценность + CTA
- CTA: «Читайте в статье», «Узнайте как», «Пошаговый разбор»
- Без кавычек и спецсимволов

Пример:
```
«Аудит отдела продаж — как найти где теряются лиды, что проверить в CRM и скриптах. Чек-лист от практиков из 200+ проектов.»
(127 символов ✓)
```

Предложи **2 варианта**.

## Шаг 4 — Проверка H2

Каждый H2 должен быть самостоятельным поисковым запросом. Проверь список H2 в статье:

| H2 | Статус | Правка |
|----|--------|--------|
| [текущий H2] | ✅ / ❌ | [предложение если ❌] |

Запрещённые паттерны H2:
- «Что такое X» (информационный мусор)
- «Почему X важен» (очевидно)
- «Итог» / «Заключение» (не поисковый запрос)
- Абстрактные: «Как это работает», «Ключевые моменты»

## Шаг 5 — FAQ

Минимум 4 вопроса. Источники:
1. WebSearch «[ключ] люди также спрашивают» → Яндекс
2. WebSearch «[ключ] site:wordstat.yandex.ru» или автосаджест
3. Вопросы, на которые статья уже отвечает, но без явного H2

Формат вывода:
```
Q: [вопрос буквально из поиска]
A: [2-4 предложения, конкретный ответ]
```

## Шаг 6 — Обложка (cover_subtitle + cover_tag)

**cover_subtitle** — подзаголовок обложки, 3–5 слов строчными:
- Описывает угол статьи: «как выявить слабые места», «чек-лист и методология»
- НЕ повторяет H1 дословно

**cover_tag** — тег на обложке, 2–4 слова ЗАГЛАВНЫМИ через «·»:
- Жанр + год: «ПОШАГОВЫЙ ГАЙД · 2025», «ЧЕК-ЛИСТ · ПРИМЕРЫ»
- Не более 4 слов

Предложи 2 варианта каждого.

## Шаг 7 — Внутренние ссылки

Проверь: есть ли в статье 3–5 внутренних ссылок на страницы Фактора.

Опорная карта (`factor/data/company-context.md` раздел 9):
- Платный аудит → `/services/platniy-audit/`
- Построение ОП → `/services/postroenie-otdela-prodazh/`
- Кейсы → `/case/`
- Блог (смежные статьи) → из `company-context.md`

Если ссылок меньше 3 — предложи конкретные места вставки с анкором.

---

## Gotchas

- **Считать символы явно** — «~50 символов» не годится. Считать точно.
- **H1 ≠ SEO Title** — разные тексты. H1 на странице, SEO Title в браузерной вкладке и сниппете.
- **FAQ из реального поиска, не придуманные** — использовать WebSearch для проверки реальных вопросов.
- **cover_tag — ЗАГЛАВНЫМИ** — это технические требования темы WP.
- **Не переписывать тело статьи** — только метаданные и SEO-оболочка.
