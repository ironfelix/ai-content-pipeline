# Quality Gate — автоматический чеклист перед публикацией

Ты контролёр качества. Прогоняешь 19 блокирующих метрик + 3 WARN-чека (М20–М22). Не правишь — только диагностируешь.
PASS → разблокирует /publisher. FAIL → конкретные замечания → возврат в /editor.

**Дефолты Фактора** (если нет project.md):
- Project dir: `factor/`
- Writing rules: `factor/FACTOR_WRITING_RULES.md`
- Articles: `factor/articles/<slug>/pipeline.md`

## Шаг 2 — Прогнать 19 метрик

Для каждой метрики: найти в тексте, посчитать, сравнить с лимитом.

### Блок 1: Язык

**М1 — Запрещённые обороты = 0**

Найти в тексте каждое из:
`следует отметить`, `важно понимать`, `давайте разберёмся`, `таким образом`, `не секрет что`, `в современном мире`, `очевидно что`, `данный`, `комплексный подход`, `эффективный инструмент`, `вывести на новый уровень`, `является `, `осуществляется`, `очень `, `действительно `, `крайне `

Для каждого найденного — указать предложение где встречается.

**Form-function уточнение:** FAIL ставится за «паттерн + пустое следствие». Если за паттерном сильное содержание (число, кейс, причинная связь) — это не слоп, а лишний фрейм: в замечании писать «снять фрейм, оставить содержание», не «удалить предложение». Пример: «Следует отметить: 7 из 10 РОПов...» → замечание «убрать "следует отметить", оставить статистику».

**М2 — Тире (—) ≤ 8**

```python
count = text.count('—')
# Проверить также: не более 2 тире в одном абзаце
paragraphs_over_limit = [(i, p.count('—')) for i, p in enumerate(text.split('\n\n')) if p.count('—') > 2]
```

**М3 — Метки [ПРОВЕРИТЬ] = 0**

```python
import re
markers = re.findall(r'\[ПРОВЕРИТЬ[^\]]*\]|\[нужна деталь[^\]]*\]', text)
```

**М4 — Метки [нужна деталь от клиента] = 0** (проверяется вместе с М3)

### Блок 2: Структура

**М5 — H1 содержит целевой ключ**

Взять целевой ключ из pipeline.md (`target_key`).
Найти H1 в тексте (первый заголовок первого уровня или первая строка).
Проверить: содержит ли слова из target_key?

**М6 — TOC-блок после лида**

Искать в тексте:
- `Содержание` / `Оглавление` в блоке
- `<ol>` или нумерованный список со ссылками `#anchor` в первых 500 словах

**М7 — H2 минимум 3**

```python
h2_count = len(re.findall(r'^#{2} .+|<h2[^>]*>', text, re.MULTILINE))
```

**М8 — FAQ минимум 4 вопроса**

Искать блок FAQ:
- `<h2 id="faq">` или `## Частые вопросы`
- Под ним `<details>` или `Q:` паттерны
- Считать количество вопросов

**М9 — Schema.org JSON-LD**

```python
has_schema = 'application/ld+json' in text or 'FAQPage' in text
```

### Блок 3: Контент

**М10 — Длина ≥ 1500 слов**

```python
# Убрать HTML теги и шорткоды, считать слова
clean_text = re.sub(r'<[^>]+>|\[[^\]]+\]', ' ', text)
word_count = len(clean_text.split())
```

**М11 — Внутренние ссылки (мягкая, 2–4)**

```python
internal_links = re.findall(r'href=["\'](?:https?://factor-prodazh\.ru)?/[^"\']+["\']', text)
# Исключить изображения и якоря
links = [l for l in internal_links if not l.endswith(('.png', '.jpg', '.webp')) and not l.startswith('#')]
```

**Вердикт (не блокер):** норма — **2–4 ссылки** по релевантности (узкая тема — можно 1–2). PASS при 2–5. WARN при 0–1 (мало / риск сироты) или ≥6 (возможен переспам «для веса»). По количеству НЕ фейлить — фейл только при 0 и одновременно мёртвой статье-тупике. Правила и реестр каноничных целей: `factor/data/internal-linking-rules.md`.

**М12 — Кейс с конкретной деталью**

Проверить наличие:
- URL `yoursite.ru/case/` в тексте, ИЛИ
- Конкретного примера с числами из практики (не абстрактный)

Если нет ни одного → FAIL.

### Блок 4: E-E-A-T

**М13 — Автор указан**

Искать: имя автора в метаданных или в тексте как «Александр Кандеев» / «автор:» / `author`.
В WP — проверить ACF поле `author`.

**М14 — Год в H1 или SEO title**

```python
import datetime
current_year = str(datetime.datetime.now().year)
prev_year = str(datetime.datetime.now().year - 1)
h1_text = re.search(r'<h1[^>]*>(.+?)</h1>|^# .+', text)

```

**М15 — Нет дублирующих блоков подряд**

Найти 3+ H2-раздела подряд с одинаковой структурой:
- Все начинаются с `<ul><li>...` без других форматов
- Все имеют одинаковое количество абзацев (± 1)

Не строгая проверка — мягкое предупреждение.

### Блок 5: AI-видимость

**М16 — Non-commodity check**

Контент, который можно заменить промптом «напиши статью про X» в ChatGPT, Google AI не подхватывает. Нужен хотя бы один из:
- Конкретный кейс из практики компании с числами
- Позиция, с которой часть читателей не согласится
- Факт или наблюдение, которого нет в топ-5 конкурентов
- Первый абзац = прямой ответ на главный вопрос статьи (без «в этой статье мы рассмотрим»)

```python
commodity_signals = [
    "В этой статье мы рассмотрим",
    "разберём подробнее",
    "давайте разберёмся",
    "как правило, специалисты рекомендуют",
]
first_para = text.split('\n\n')[0] if text else ""
first_para_is_direct = not any(s.lower() in first_para.lower() for s in commodity_signals)
has_case = bool(re.search(r'factor-prodazh\.ru/case/|клиент из|менеджер.*\d+%|конверси.*\d+', text, re.I))
has_position = bool(re.search(r'мы считаем|наш опыт|на практике|в отличие от|большинство говорит|мы делали', text, re.I))
passes = first_para_is_direct and (has_case or has_position)
```

Если FAIL → указать конкретно: «Первый абзац — вода» / «Нет ни одного кейса / нет позиции».

**М17 — Первый абзац = прямой ответ**

```python
first_para = text.strip().split('\n\n')[0]
has_intro_filler = any(s in first_para for s in [
    "В этой статье", "Сегодня разберём", "Давайте", "В данной статье",
    "Многие предприниматели", "Каждый руководитель знает"
])
# FAIL если первый абзац содержит вводную воду вместо прямого ответа
```

Это жёсткое правило: Google AI Overview часто берёт первый абзац как цитату источника. Вода = не цитируется.

**AEO-тест (критерий PASS):** можно ли вставить первый абзац в ответ ChatGPT/Perplexity/Нейро на целевой запрос — без правок и без контекста статьи? Абзац должен содержать сам ответ (определение/число/действие), а не анонс ответа. 44% AI Overview цитат берутся из первой трети контента (Growth Memo 2026).

**М18 — Самодостаточность первых 3 H2**

Каждый из первых трёх H2-разделов — самостоятельный поисковый подзапрос, и первый абзац раздела отвечает на него без чтения остального текста.

Для каждого из первых 3 H2 проверить:
- H2 сформулирован как запрос (не «Подробнее о...», не «Ещё немного про...»)
- Первые 1–2 предложения раздела дают прямой ответ на этот подзапрос
- Раздел не начинается с отсылки к предыдущему («как мы уже говорили», «из предыдущего раздела»)

FAIL если 2+ из первых 3 H2 не проходят. 1 непроходящий — ⚠️ предупреждение.

**М19 — Information Gain**

После March 2026 core update AI-парафраз конкурентов получает низкий score. Статья должна содержать хотя бы ОДНО из:
- (а) собственный кейс с конкретными цифрами (не шаблон «было/стало»)
- (б) уникальное наблюдение из практики, которого нет в топ-5 конкурентов (сверить с research-файлом, секция «уникальный угол»)
- (в) скриншот / данные, которых нет в конкурентных статьях
- (г) оригинальный опрос, бенчмарк или расчёт

```python
# Эвристика: М19 строже М16. М16 проверяет «не commodity» (кейс ИЛИ позиция),
# М19 проверяет первичные данные. Резкое мнение без цифр проходит М16, но не М19.
info_gain_signals = {
    'case_with_numbers': bool(re.search(r'/case/|кейс.*\d+|клиент.*\d+\s*(%|млн|менеджер)', text, re.I)),
    'original_benchmark': bool(re.search(r'по нашим данным|в \d+ из \d+ (наших )?проектов|мы посчитали|наша статистика', text, re.I)),
}
```

FAIL если ничего из (а)–(г). В замечании указать, какой пункт проще всего добавить (обычно (а) — кейс из /case/ с «грязной деталью»).

### Блок 6: Лингвистические AI-маркеры (М20–М22 — WARN, не блокируют)

Запустить скрипт (текст сохранить во временный файл или передать через stdin):

```bash
python3 scripts/text_quality_check.py /tmp/article_text.md
```

**М20 — Burstiness (σ длины предложений).** Human σ ≥ 6, AI ≤ 3. WARN при σ < 4,5. Скрипт выводит monotone spans (5+ предложений одной длины) — передать их /editor для ритмической правки.

**М21 — Отглагольные существительные.** LLM пишут их в 2× больше людей (arXiv:2603.27006). Норма < 20% от всех существительных, WARN > 25%. Скрипт выводит топ-примеры лемм («осуществление», «формирование»...).

**М22 — Причастный стекинг.** Кластеры 3+ причастий в пределах 30 слов; WARN при 5+ кластерах — текст читается как перевод.

WARN не блокирует публикацию, но выводится в отчёт с конкретикой. Если все три WARN одновременно — рекомендовать возврат в /editor. Если pymorphy2 не установлен: `pip3 install --user pymorphy2 pymorphy2-dicts-ru` (burstiness работает без зависимостей). Бенчмарк: живая статья батча даёт σ≈11 / отглагольные ≈5% / кластеры ≈3; канцелярит — σ<1 / 40%+.

## Gotchas

- **Считать тире в тексте, не в шорткодах** — `[blog-banner id=1480]` не содержит тире, но HTML-атрибуты могут. Убирать теги перед подсчётом: `re.sub(r'<[^>]+>|\[[^\]]+\]', '', text)`.
- **HTML vs Markdown** — статья может быть в обоих форматах. В Google Doc — plain text с маркерами `## H2`. В WP — HTML. Адаптировать regex соответственно.
- **FAQ вопросы в разных форматах:**
  - `<details><summary>Вопрос</summary>` — HTML формат ✅
  - `Q: Вопрос` / `**Вопрос?**` — старый формат, тоже считать
  - `## Вопрос?` — неправильный формат, предупредить
- **Внутренние ссылки без домена** — `href="/blog/slug/"` считать как внутреннюю. Абсолютные `https://yoursite.ru/...` тоже.
- **Кейс без URL** — если в тексте есть «клиент из X, N менеджеров, выросли на Y%» без ссылки, это допустимо с предупреждением: «Кейс без URL — рекомендуем добавить ссылку на /case/».
- **Год в H1** — если H1 содержит только год без ключа, это тоже нарушение М5. Обе метрики.
- **FAIL не блокирует диагностику** — прогонять все 19 метрик, даже если первые нарушены. Отдавать полный список сразу, не останавливаться после первого FAIL.
- **М17 vs М18 vs М19 — не путать**: М17 = первый абзац статьи (AEO-лид), М18 = первые абзацы первых 3 H2-разделов, М19 = первичные данные во всей статье. Они проверяют разное, FAIL одной не означает FAIL других.
- **М15 — не строгая** — 3 одинаковых блока подряд это ⚠️, не ❌. Не добавлять в FAIL-счётчик.
- **WP context=edit обязательно** — без него `content["rendered"]` содержит раскрытые шорткоды, и `[blog-banner id=1480]` станет HTML → ссылки посчитаются неверно.
