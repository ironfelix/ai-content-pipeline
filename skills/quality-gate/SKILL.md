# Quality Gate — автоматический чеклист перед публикацией

Ты контролёр качества. Прогоняешь 17 метрик. Не правишь — только диагностируешь.
PASS → разблокирует /publisher. FAIL → конкретные замечания → возврат в /editor.

**Дефолты Фактора** (если нет project.md):
- Project dir: `factor/`
- Writing rules: `factor/FACTOR_WRITING_RULES.md`

### Блок 5: AI-видимость

**М16 — Non-commodity check**

Контент, который можно заменить промптом «напиши статью про X» в ChatGPT, Google AI не подхватывает. Нужен хотя бы один из:
- Конкретный кейс из практики компании с числами
- Позиция, с которой часть читателей не согласится
- Факт или наблюдение, которого нет в топ-5 конкурентов
- Первый абзац = прямой ответ на главный вопрос статьи (без «в этой статье мы рассмотрим»)

```python
commodity_signals = [
    "В этой статье мы рассмотрим",
    "разберём подробнее",
    "давайте разберёмся",
    "как правило, специалисты рекомендуют",
]
first_para = text.split('\n\n')[0] if text else ""
first_para_is_direct = not any(s.lower() in first_para.lower() for s in commodity_signals)
has_case = bool(re.search(r'factor-prodazh\.ru/case/|клиент из|менеджер.*\d+%|конверси.*\d+', text, re.I))
has_position = bool(re.search(r'мы считаем|наш опыт|на практике|в отличие от|большинство говорит|мы делали', text, re.I))
passes = first_para_is_direct and (has_case or has_position)
```

Если FAIL → указать конкретно: «Первый абзац — вода» / «Нет ни одного кейса / нет позиции».

**М17 — Первый абзац = прямой ответ**

```python
first_para = text.strip().split('\n\n')[0]

    "В этой статье", "Сегодня разберём", "Давайте", "В данной статье",
    "Многие предприниматели", "Каждый руководитель знает"

# FAIL если первый абзац содержит вводную воду вместо прямого ответа
```

Это жёсткое правило: Google AI Overview часто берёт первый абзац как цитату источника. Вода = не цитируется.

- Все начинаются с `<ul><li>...` без других форматов
- Все имеют одинаковое количество абзацев (± 1)

## Gotchas

- **Считать тире в тексте, не в шорткодах** — `[blog-banner id=1480]` не содержит тире, но HTML-атрибуты могут. Убирать теги перед подсчётом: `re.sub(r'<[^>]+>|\[[^\]]+\]', '', text)`.
- **HTML vs Markdown** — статья может быть в обоих форматах. В Google Doc — plain text с маркерами `## H2`. В WP — HTML. Адаптировать regex соответственно.
- **FAQ вопросы в разных форматах:**
  - `<details><summary>Вопрос</summary>` — HTML формат ✅
  - `Q: Вопрос` / `**Вопрос?**` — старый формат, тоже считать
  - `## Вопрос?` — неправильный формат, предупредить
- **Внутренние ссылки без домена** — `href="/blog/slug/"` считать как внутреннюю. Абсолютные `https://YOUR_DOMAIN.ru/...` тоже.
- **Кейс без URL** — если в тексте есть «клиент из X, N менеджеров, выросли на Y%» без ссылки, это допустимо с предупреждением: «Кейс без URL — рекомендуем добавить ссылку на /case/».
- **Год в H1** — если H1 содержит только год без ключа, это тоже нарушение М5. Обе метрики.
- **FAIL не блокирует диагностику** — прогонять все 15 метрик, даже если первые нарушены. Отдавать полный список сразу, не останавливаться после первого FAIL.
- **М15 — не строгая** — 3 одинаковых блока подряд это ⚠️, не ❌. Не добавлять в FAIL-счётчик.
- **WP context=edit обязательно** — без него `content["rendered"]` содержит раскрытые шорткоды, и `[blog-banner id=1480]` станет HTML → ссылки посчитаются неверно.
